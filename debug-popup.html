🔧 NavSphere 扩展调试工具

⚠️ 调试说明

这个页面帮助你调试 NavSphere 扩展的"提示成功但没提交上"问题。

请按照以下步骤进行调试：

步骤 1: 检查扩展状态

首先确认扩展已正确安装并启用。

步骤 2: 测试快速添加功能

在这个页面上测试快速添加功能，观察控制台输出。

步骤 3: 查看控制台日志

打开浏览器开发者工具 (F12)，查看 Console 标签页的输出。

特别关注以下关键日志：

  • 🚀 开始添加书签 - 确认添加流程开始
  • 🔐 认证状态: true - 确认认证成功
  • 📡 调用API添加书签... - 确认API调用开始
  • ✅ API调用成功 - 确认API调用成功
  • ❌ 添加书签失败 - 如果出现错误

步骤 4: 检查网络请求

在开发者工具的 Network 标签页中，查看是否有对 NavSphere API 的请求。

查找类似这样的请求：POST /api/navigation/{categoryId}/items

网络请求调试步骤：

  1. 打开开发者工具 (F12)
  2. 切换到 "Network" 标签页
  3. 确保 "Preserve log" 选项已勾选
  4. 尝试添加书签
  5. 查看是否有对你的 NavSphere 实例的 POST 请求
  6. 点击请求查看详细信息：
    • Request Headers - 检查 Authorization 头是否存在
    • Request Payload - 检查发送的数据是否正确
    • Response - 检查服务器返回的响应

步骤 5: 常见问题排查

🚨 可能的问题原因：

  • 认证问题：实例未完成 GitHub OAuth 认证
  • 网络问题：CORS 策略阻止了跨域请求
  • API 端点问题：NavSphere 实例的 API 端点不正确或不可用
  • 数据验证失败：发送的书签数据不符合服务器要求
  • 服务器错误：NavSphere 服务器内部错误

✅ 解决方案：

  • 确保在扩展设置页面完成了 GitHub OAuth 认证
  • 检查 NavSphere 实例的 URL 是否正确且可访问
  • 确认 NavSphere 服务器正在运行且 API 可用
  • 检查浏览器是否阻止了跨域请求
  • 查看 NavSphere 服务器的日志文件
